WHAT IS CONCURRENCY AND HOW CAN IT BE APPLIED INTO UNITY?

  • Research done by:Marc Alcón Meliá&Cristina García Armijo
  • Finish date of the research:January 10, 2021
  • Duration and experimentation:4 months and a half
  • Can be found here:Unity project + PDF Document

Either in computer science or when programming games, we as programmers may found ourselves sometimes in the situation that the game that we are building runs slow because we added too many tasks that deteriorate the performance of the game, and in those cases one good solution that we can apply is concurrency, which will make the game run significantly faster.

In this research article, first it is explained what concurrency is and why is it useful, and then it gets a bit more complex, because it starts to explain in which ways can concurrency be implemented into Unity, either using Coroutines, Jobs or Threads, and it finishes giving a conclusion about which application is better in each case and justifies it.

It's kind of a divide and conquer approach to perform the same tasks but in an optimized way

Additional to the research article, there is also a Unity project, that displays a maze with several capsule objects, ones that use concurrency (we used threads in the project) and other that don't, and it can clearly be seen how they go faster and together, rather than slower and one after another because of the no use of concurrency in them.

Aside from that, it also implements an A* algorithm to find the best solution of the maze and guide the capsules to the goal.
